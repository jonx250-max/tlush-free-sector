04 · LEGAL PROTECTION

הגנה משפטית — Audit Trail

לכל בדיקה נשמר snapshot מדויק של גרסת החוק שהשתמשה בה. אם משתמש טוען "המערכת טעתה" — נוכל להראות בדיוק מה היה הקלט שלו, איזו גרסה של איזה חוק שימשה, והקישור למקור הרישמי.

לכל בדיקה נשמר: 
{
  "check_id": "chk_2026_12_09_abc123",
  "timestamp": "2026-12-09T14:32:18Z",
  "input": { ... payslip + tax profile ... },
  "laws_snapshot": {
    "version": "2026.12.09",
    "git_commit": "a3f2c8d",
    "snapshot_url": "data/laws/_versions/2026-12-09.tar.gz",
    "laws_used": [
      "credit-points-single-parent-2026@v3",
      "minimum-wage-2026@v1"
    ]
  },
  "result": { ... },
  "disclaimer_version": "v2.4"

05 · EDGE CASES

מקרי קצה שטופלו

Scraper נשבר (אתר שונה מבנה)
אזעקה למייל אדמין, ממשיך עם השאר. retry 3x לפני skip.
AI classifier טעה
confidence < 0.85 → manual review queue במייל יומי לאדמין.
חוק רטרואקטיבי
flag מיוחד, מייל דחוף, re-evaluate היסטוריה אוטומטי.
חוק שמתבטל
effective_until ב-JSON, גרסה ישנה נשמרת ב-snapshot.
תאריך לא ברור
confidence יורד אוטומטית, נכנס ל-manual review.
שינוי שעלול לפגוע במשתמשים
dry-run, השוואה ל-baseline, אישור ידני לפני push.
